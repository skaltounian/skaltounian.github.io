Current Project

I recently learned to modularise my pages. I just broke out my footer content into a new file, and am using a js to fetch the content into this and other pages.

By the time this is done, I will have:

  1. Use the fetch command to read data from another file. ✓
  2. Use the document.getElementById to insert the data in the intended location. ✓
  3. Create an external footer file that is common to multiple pages. ✓
  4. Move the menu content to an external file and reuse in multipe pages. ✓
  5. Sort out page hirearchy and link paths in the menu, breadcrumb and parent pages. ✓
  6. Move the js scripts from pages to another in the scripts folder
  7. Reuse the scripts for multiple imports - still have to learn how to do this.

Local Server

While implementing this feature, I learned that when working with fetch() on local files, the browser may block the request due to security restrictions. This means fetch() will not work when directly opening an HTML file via file://.

The solution was to set up a local server on my computer, then test again. And it worked!!
